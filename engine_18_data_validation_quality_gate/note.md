# NOTE — DAY 1

## Pemahaman Hari Ini
Engine ini bukan soal teknis validation,
tapi soal keberanian mengambil keputusan terhadap data.

## Insight Penting
Jika data buruk tetap diproses,
maka semua analisis setelahnya adalah kebohongan yang rapi.

## Pertanyaan untuk Diri Sendiri
- Kapan data harus dihentikan?
- Apakah semua kesalahan data bisa ditoleransi?

## Catatan untuk Masa Depan
Engine ini adalah rem, bukan mesin gas.

# NOTE — DAY 2

## Pemahaman Hari Ini
Validation rule bukan soal benar atau salah,
tapi soal batas risiko yang masih bisa diterima.

## Insight Penting
Rule keras melindungi sistem.
Rule lunak melindungi fleksibilitas bisnis.

## Pertanyaan untuk Diri Sendiri
- Rule mana yang berani aku jadikan FAIL?
- Apakah aku menulis rule ini sebagai engineer atau tukang script?

## Catatan untuk Masa Depan
Jika semua rule warning,
berarti aku takut mengambil keputusan.

# NOTE — DAY 3

## Pemahaman Hari Ini
Validator bukan hakim tunggal.
Ia saksi yang memberi fakta ke Quality Gate.

## Insight Penting
Missing value dan schema error
adalah jenis kerusakan yang paling sering disembunyikan.

## Pertanyaan untuk Diri Sendiri
- Kenapa schema error harus FAIL cepat?
- Apakah missing selalu berarti data buruk?

## Catatan untuk Masa Depan
Jika validator terlalu pintar,
engine akan kehilangan kendali.
