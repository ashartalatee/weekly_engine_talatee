## Data Snapshot Engine

### Tujuan
Snapshot dibuat untuk:
- mengetahui kondisi data objektif
- membandingkan versi tanpa membuka file manual
- menjadi dasar laporan ke klien

### Insight Hari Ini
Snapshot bukan backup.
Snapshot adalah "hasil audit singkat".

Angka-angka seperti:
- row count
- missing values
- duplicate rows

lebih berbicara daripada ribuan baris data mentah.

### Kesalahan Umum
- hanya menyimpan file
- tidak mencatat metadata
- tidak menyimpan sample data

Tanpa snapshot:
versi data tidak bisa dibandingkan secara adil.
