# Engine 02 — Missing & Null Pattern
Hari 1 — Fondasi

## Kenapa engine ini dibuat?
Karena klien sering tidak sadar:
- Data mereka rusak
- Laporan mereka salah
- Keputusan mereka bias

Missing bukan sekadar nilai kosong.
Missing adalah SINYAL proses bisnis bermasalah.

## Tujuan engine ini
Engine ini TIDAK membersihkan data.
Engine ini membuka mata:
- Di kolom mana data sering kosong
- Di baris seperti apa missing muncul
- Apakah missing terjadi acak atau berpola

## Perubahan cara berpikir saya
❌ Dulu: "Berapa banyak data kosong?"
✅ Sekarang: "KENAPA data ini kosong?"

## Output yang saya inginkan
- Ringkasan missing per kolom
- Pola missing (bersamaan / waktu / kategori)
- Insight yang bisa dijelaskan ke klien non-teknis

## Prinsip kerja
1. Fakta dulu, solusi belakangan
2. Jangan membersihkan sebelum paham
3. Engine > Script

Hari ini saya belum menulis kode.
Hari ini saya menanam fondasi berpikir.

---

## Hari 2 — Missing adalah Keputusan

Hari ini saya sadar:
Missing bukan urusan teknis,
tapi urusan definisi & konteks bisnis.

Contoh:
- Angka 0 bisa valid
- String "NA" bisa berarti rusak
- Kolom tertentu punya standar berbeda

Dengan rule ini:
- Saya tidak debat dengan klien
- Saya tunjukkan definisinya
- Mesin mengikuti keputusan

Ini mengubah posisi saya:
dari "yang ngoding"
menjadi "yang menetapkan standar".

## Hari 3 — Fakta Dulu, Insight Belakangan

Hari ini engine saya:
- Tidak pintar
- Tidak memberi saran
- Hanya berkata jujur

Saya belajar menahan diri:
tidak langsung menyimpulkan.

Angka missing hanyalah bahan baku.
Nilai sesungguhnya akan muncul
saat saya melihat polanya.

## Hari 4 — Pola Mengubah Segalanya

Hari ini saya berhenti bertanya:
"Berapa banyak data kosong?"

Saya mulai bertanya:
"Data seperti APA yang kosong?"

Saya sadar:
- Missing jarang acak

Di titik ini,
saya tidak lagi membersihkan data,
saya menyelidiki proses bisnis.

## Hari 5 — Nilai Saya Ada di Cerita

Hari ini saya sadar:
kode tanpa laporan = tidak bernilai.

Saya belajar menerjemahkan:
- angka → cerita
- logika → risiko
- missing → dampak bisnis

Di titik ini,
saya bukan lagi sekadar programmer.
Saya penghubung data dan keputusan.
